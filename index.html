Project 1: Investment Company System (Software Design)

This started as code from a reverse engineering assignment in CS 405. The original had global variables everywhere and was really hard to follow.

Implemented the Singleton design pattern to clean everything up. Instead of having name1, name2, var1, var2 etc scattered around, I created a proper class structure.

// Original code
string name1 = "Bob Jones";
string name2 = "Sarah Davis";
int var1 = 1;  // service choice
int var2 = 1;

// enhancement
class InvestmentCompany {
private:
    static InvestmentCompany* instance;
    struct Client {
        string name;
        int serviceOption;
    };
    Client clients[5];
};

I can take bad code and refactor it using proper design patterns. The singleton ensures only one instance exists which makes sense for this system.

Project 2: Collection Testing Suite (Data Structures)

Had a Google Test suite that tested std::vector operations. Needed to make it work with std::array instead.

Arrays have fixed size so you cannot just push_back() like with vectors

// replace vector with std:array 
template
class CollectionTest : public ::testing::Test {
    std::unique_ptr> collection;
    size_t element_count; 
    
    void add_entries(int count) {
        // no longer using vector, so cant use push_back
        for (int i = 0; i < count; i++) {
            (*collection)[element_count++] = rand() % 100;
        }
    }
};

The data structure to be used really depends on goal time complexeties and size of the data we are processing.

Project 3: SQL Injection Defense (Database Security)

Original code from CS 405 only checked for "OR" to prevent SQL injection

  • Added detection for common attack patterns (1=1, --, /**/)
  • Implemented input validation
  • Used prepared statements 
// use prepared statements
bool run_safe_query(sqlite3* db, const string& username) {
    sqlite3_stmt* stmt;
    const char* sql = "SELECT * FROM USERS WHERE NAME = ?";
    
    sqlite3_prepare_v2(db, sql, -1, &stmt, 0);
    sqlite3_bind_text(stmt, 1, username.c_str(), -1, SQLITE_STATIC);
    
  
}

A more secure approach to protect against real SQL injection attempts
